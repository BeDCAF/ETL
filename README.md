**План работы:**

**Изучение структуры ELT (формат данных, особенности хранения в Windows).**

**Реализация чтения данных из файлов ELT с обработкой событий.**

**Оптимизация структуры для вывода в удобный формат (CSV или JSON).**

**Добавление выбора формата через аргументы командной строки.**

**Таблица описания структуры elt файла**

| **Колонка**             | **Тип данных**    | **Описание**                                                                 |
|-------------------------|-------------------|----------------------------------------------------------------------------- |
| **Event Name**          | string            | Имя события (обычно `<ProviderName>`)                                        |
| **Type**                | string/число      | Тип события/канала (например, Header, Data, иногда числовой идентификатор)   |
| **Event ID**            | число             | Числовой идентификатор события                                               |
| **Version**             | число             | Версия определения события                                                   |
| **Channel**             | число             | Канал (Channel): 0 = None, 1 = Application, 2 = System, 16 = Security и т.д. |
| **Level**               | число             | Уровень (Level): 0 = Log Always, 1 = Critical, 2 = Error, 3 = Warning и т.д. |
| **Opcode**              | число             | Код операции внутри события (обычно 0)                                       |
| **Task**                | число             | Идентификатор задачи (обычно 0)                                              |
| **Keyword**             | hex               | Битовая маска ключевых слов (Keyword)                                        |
| **PID**                 | hex               | Process ID — ID процесса, записавшего событие                                |
| **TID**                 | hex               | Thread ID — ID потока                                                        |
| **Processor Number**    | число             | Логический номер процессора                                                  |
| **Instance ID**         | число             | Идентификатор экземпляра события при множественных сессиях                   |
| **Parent Instance ID**  | число             | Идентификатор родительского экземпляра (если есть)                           |
| **Activity ID**         | GUID              | GUID активности, позволяет коррелировать цепочку событий                     |
| **Related Activity ID** | GUID              | Связанный Activity ID (если есть)                                            |
| **Clock-Time**          | 64-бит int        | Метка времени в формате Windows FILETIME (100-нс тики от 1 января 1601)      |
| **Kernel(ms)**          | число (мс)        | Время в ядре до момента события (при включённой трассировке ядра)            |
| **User(ms)**            | число (мс)        | Время в режиме пользователя                                                  |
| **User Data**           | string/HEX        | Полезная нагрузка события: текст (если читаемо) или HEX-строка               |
